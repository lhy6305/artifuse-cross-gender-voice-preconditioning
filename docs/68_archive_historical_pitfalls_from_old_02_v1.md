# 归档：旧版 02 的历史坑点摘出

## 说明

本文件从旧版 `docs/02_pitfalls_log.md` 中拆出，保留那些：

- 曾经真实出现过
- 但现在不再属于活跃接班必须阅读项
- 或者更偏数据集/阶段历史背景

的坑点。

## 归档的历史坑点类型

1. `VocalSet` 三套重复目录树导致重复计数
2. `VocalSet` 公共页面与包内 readme 的性别数量不一致
3. `VCTK` 本地音频是成对 `mic1/mic2` 的 FLAC
4. 数据集下载包与 `raw/` 树的忽略边界细节
5. 关键 checkpoint 的白名单策略形成过程
6. 某个下载包曾被 Git 跟踪的历史纠偏
7. 根目录本地运维脚本迁移过程
8. `.editorconfig` 引入前后的过程性问题

## 当前规则

- 活跃 `docs/02_pitfalls_log.md` 只保留仍然有效的全局坑点
- 数据集级、阶段级、已解决的历史问题改放归档
- 若某条历史坑点重新变成活跃风险，再迁回活跃文档并改成英文 ASCII

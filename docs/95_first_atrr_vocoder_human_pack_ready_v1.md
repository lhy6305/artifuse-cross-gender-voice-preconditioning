## Build Method

The pack was not assembled manually.

It is now generated by:

- `scripts/build_atrr_vocoder_human_review_pack.py`

That script reuses the old fixed8 queue as the row-order and metadata template,
then swaps in:

- the new carrier output as `processed_audio`
- refreshed machine-side `auto_*` fields from the active adapter summary

This keeps the human review format stable while allowing future rebuilds from a
new carrier summary without hand-editing CSV files.

## 关键结论

- 当前结果足以说明项目已经进入可分析阶段。
- 当前结果还不足以直接写出修正规则。
- 下一轮应优先推进 `stage0 baseline full`，而不是跳过分析直接进入模型实现。
- `full` 不必再作为一次性长跑任务处理，可以拆成 `speech -> singing -> finalize` 三步执行。
- `full` 已经确认 `speech` 侧方向更稳定，但跨数据集绝对分布差异仍大。
- `full` 已经确认 `singing` 侧 style 条件效应很强，后续不能把不同 style 直接折叠。
- 当前 `f0` 全局四分位分桶会产生若干极稀疏 cell，下一轮应优先改进分桶方式。
- 稳健分桶现在已改为按 dataset/style 自适应选 `quartile / tertile / median_split`，并以“可比较 bin 数最大化”为准。
- `forte` 与 `pp` 当前只存在中间窄重叠区可比较，说明这些 style 的规则不能按常规双边分桶直接展开。
- 第一版候选先验应先收缩到 `spectral centroid` 与 `log_centroid_minus_log_f0` 主线。
- `f0` 相关字段继续保留为条件变量，不直接当 correction target。
- `rms_dbfs`、`silence_ratio_40db`、`f0_voiced_ratio` 当前更适合作为分析监控或清洗参考。
- 第一版方向性规则当前应先只在 `singing` 域的稳定 style 上保守启用。
- `speech` 侧继续保留观察态，不直接写入统一规则。
- 当前已经有可直接读取的 CSV / JSON 候选规则配置，可直接供最小前置器原型消费。
- selector 原型已经确认当前规则边界生效：`forte/pp` 无误命中，`median_split` 型 style 基本按 `50%` 样本覆盖工作。
- `brightness_up / brightness_down` 已映射到 6 粗频带的保守 gain 模板，当前已经具备最小 band-gain 原型。
- 最小规则前置器原型已跑通，试听包中的处理音频与原音频存在可测差异，不是空操作。
- 量化评审结果显示：当前原型整体保真代价低，但代理指标推动仍偏弱，第一轮更像“安全但不够到位”。
- 规则试听量化构建首轮仍需约半分钟，但二次打开已可复用缓存，重建耗时降到约 `2.3s`。

## 关键结论

- 当前结果足以说明项目已经进入可分析阶段。
- 当前结果还不足以直接写出修正规则。
- 下一轮应优先推进 `stage0 baseline full`，而不是跳过分析直接进入模型实现。
- `full` 不必再作为一次性长跑任务处理，可以拆成 `speech -> singing -> finalize` 三步执行。
- `full` 已经确认 `speech` 侧方向更稳定，但跨数据集绝对分布差异仍大。
- `full` 已经确认 `singing` 侧 style 条件效应很强，后续不能把不同 style 直接折叠。
- 当前 `f0` 全局四分位分桶会产生若干极稀疏 cell，下一轮应优先改进分桶方式。
- 稳健分桶现在已改为按 dataset/style 自适应选 `quartile / tertile / median_split`，并以“可比较 bin 数最大化”为准。
- `forte` 与 `pp` 当前只存在中间窄重叠区可比较，说明这些 style 的规则不能按常规双边分桶直接展开。
- 第一版候选先验应先收缩到 `spectral centroid` 与 `log_centroid_minus_log_f0` 主线。
- `f0` 相关字段继续保留为条件变量，不直接当 correction target。
- `rms_dbfs`、`silence_ratio_40db`、`f0_voiced_ratio` 当前更适合作为分析监控或清洗参考。
- 第一版方向性规则当前应先只在 `singing` 域的稳定 style 上保守启用。
- `speech` 侧继续保留观察态，不直接写入统一规则。
- 当前已经有可直接读取的 CSV / JSON 候选规则配置，可直接供最小前置器原型消费。
- selector 原型已经确认当前规则边界生效：`forte/pp` 无误命中，`median_split` 型 style 基本按 `50%` 样本覆盖工作。
- `brightness_up / brightness_down` 已映射到 6 粗频带的保守 gain 模板，当前已经具备最小 band-gain 原型。
- 最小规则前置器原型已跑通，试听包中的处理音频与原音频存在可测差异，不是空操作。
- 量化评审结果显示：当前原型整体保真代价低，但代理指标推动仍偏弱，第一轮更像“安全但不够到位”。
- 规则试听量化构建首轮仍需约半分钟，但二次打开已可复用缓存，重建耗时降到约 `2.3s`。
- `v2` 把 band-gain 强度整体放大到约 `5x` 后，量化改变量明显上升，但整体仍未自动通过，说明当前静态 6 段 EQ 仍偏保守。
- `singing` 的 `v1 / v2` 两轮人工听审都给出“无可感知差异”，说明当前应优先切到 `speech-first`，而不是继续在 singing 上微调静态 EQ。
- `speech-first` 当前已经不是“全都太弱”，而是开始出现有区分度的量化结果。
- `LibriTTS-R male -> feminine` 两条已到 `borderline`，说明静态 EQ 在 speech 上至少开始具备诊断价值。
- `VCTK` 与 `LibriTTS-R` 的响应并不一致，说明下一步更像是做 dataset-specific speech profile，而不是马上冻结统一规则。
- 但人工听审已经给出更高优先级结论：`speech-first static EQ` 依然 `8/8` 无可感知差异，因此静态 6 段 EQ 当前可视为阶段性 `null result`。
- 现在不再继续微调静态 EQ，而是切到 `voiced-envelope-warp` 作为下一条更强、仍可解释的路线。
- envelope warp `v1` 已经第一次跨过“完全无感”这条线，说明这条路线比静态 EQ 更接近可用区间。
- envelope warp `v2` 的量化也明显强于 `v1`，当前最自然的下一步就是直接听 `v2`，而不是再次猜方法。
- 但 `v2` 的人工听审说明：它更容易把 `female` 侧推成不自然的窄带感，而不是真正补出 male resonance。
- 所以下一步不再继续沿 `warp` 硬推，而是改成 `broad-resonance tilt`，优先验证“更自然的共鸣倾斜”是否成立。
- resonance tilt 的人工听审又进一步说明：这条路线虽然自然，但仍然无感，当前瓶颈已经收缩到“强度与定位都不够”。
- formant anchor 原型已经跑通，但机器侧比 resonance tilt 更弱，所以它当前是探索分支，不是已验证更优解。
- formant anchor 的人工听审也给出“无感但无伪影”，至此整条轻量频谱前置家族可以做阶段性 `null result` 判断。
- 下一步不应再继续堆同类小变体，而应做阶段门选择：转更强的 source-filter / vocal-tract 路线，或把前置器收缩为安全归一化。
- `WORLD source-filter / vocal-tract morph v1` 已实际跑通，机器侧第一次明显进入高改变量区间：`avg auto_quant_score=70.44`、`avg auto_effect_score=91.07`。
- 当前这版的主要希望在 `male -> feminine`，而主要风险在 `female -> masculine` 仍可能 `wrong_direction`。
- 但人工听审已经给出更硬的结论：这版主要生成的是底噪和瞬态脏声，而不是有效的人声方向变化。
- 所以问题不再是“参数不够好”，而是 `WORLD full resynthesis` 这条实现方式本身不适合作为阶段 0 试听原型。
- 新 pivot 改成 `WORLD analysis only + voiced STFT magnitude delta`，先把重合成噪声拿掉，再看 source-filter 差分本身有没有耳感价值。
- 新的 STFT delta `v1` 机器侧结果说明：保真明显更高，但方向性接近 `0` 且多为反向，当前更像“副作用去掉了，但有效方向仍未建立”。 
- STFT delta 的人工听审再次说明：即使方法本身干净，也依然可能单独无感，因此“前置器自己是否明显可听”已经不再适合作为唯一主 gate。
- 当前已恢复到设计稿原本的正式标准：看 `raw -> RVC` 与 `preconditioned -> RVC` 的串联差异，而不是继续堆新的单独试听原型。
